Overview

Kinvey Store Protocol

A Kinvey Store is a representation of a group of backend items, stores have several operations in common to all stores:

  • Add an object/s (save* methods)
  • Update an object/s (save* methods)
  • Get an object/s (query* methods)
  • Remove an object/s (remove* methods)
  • Configure the store (configure* methods)

Additionally, stores may have store specific methods that are documented in the Store itself. You should be able to use the documentation in KCSStore to understand how to accomplish all basic tasks in Stores without needing to refer to the specific stores implementation, but you should refer to that implementation to maximize performance or to configure store specific items prior to your app’s release.

configureWithOptions:

This is the general configuration routine for a Kinvey Store

- (BOOL)configureWithOptions:(NSDictionary *)options

Parameters

options

A dictionary containing store specific options, see store documentation for details

Return Value

A BOOL value indicating whether the options were accepted by the store.

Discussion

This is the general configuration routine for a Kinvey Store

This routine is used to pass a Store specific options dictionary to the store allow the user to configure store specific items in a way that’s compatible with multiple stores.

All stores are expected to handle the following options (however they’re allowed to interpret the values in a store specific manner):

  • backend – The backend represented by this store
  • debug – Should debugging be enabled for this store

queryWithQuery:requestConfiguration:withCompletionBlock:withProgressBlock:

Query or fetch an object (or objects) in the store.

- (KCSRequest *)queryWithQuery:(id)query requestConfiguration:(KCSRequestConfiguration *)requestConfiguration withCompletionBlock:(KCSCompletionBlock)completionBlock withProgressBlock:(KCSProgressBlock)progressBlock

Parameters

query

A query to act on a store. The store defines the type of queries it accepts, an object of type [KCSQuery query] causes all objects to be returned.

requestConfiguration

defines how the client requests should be set up by default

completionBlock

A block that gets invoked when the query/fetch is “complete” (as defined by the store)

progressBlock

A block that is invoked whenever the store can offer an update on the progress of the operation.

Return Value

KCSRequest object that represents the pending request made against the store. Since version 1.36.0

Discussion

Query or fetch an object (or objects) in the store.

This method takes a query object and calls the store to provide an array of objects that satisfies the query.
